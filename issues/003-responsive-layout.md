# 반응형 레이아웃 구현

## 개요
모바일, 태블릿, 데스크톱 환경에서 최적화된 UI 제공

## 배경
현재 UI는 데스크톱 환경에 최적화되어 있습니다.
모바일 환경에서는 사이드바와 채팅 영역이 동시에 표시되어 사용성이 떨어집니다.

## 작업 내용

### 브레이크포인트
```
sm: 640px   # 모바일
md: 768px   # 태블릿
lg: 1024px  # 데스크톱
```

### 모바일 (< 768px)
1. **사이드바**
   - 기본적으로 숨김
   - 햄버거 메뉴 버튼으로 토글
   - 슬라이드 오버레이로 표시

2. **채팅방**
   - 전체 화면 사용
   - 헤더에 뒤로가기 버튼 (채팅방 목록으로)

3. **메시지 입력**
   - 하단 고정
   - 키보드 올라올 때 입력창 위치 조정

### 태블릿 (768px - 1024px)
1. **사이드바**
   - 축소된 형태로 표시 (아이콘만)
   - 호버 시 확장

2. **채팅방**
   - 나머지 공간 사용

### 데스크톱 (>= 1024px)
- 현재 레이아웃 유지 (사이드바 256px + 채팅 영역)

### 구현 세부사항

1. **AppLayout.tsx 수정**
   - `useMediaQuery` 훅 사용 또는 CSS만으로 처리
   - 모바일에서 사이드바 상태 관리

2. **Sidebar.tsx 수정**
   - 모바일 오버레이 모드
   - 축소 모드 지원

3. **ChatRoom.tsx 수정**
   - 모바일 헤더 (뒤로가기 버튼)
   - 채팅방 미선택 시 안내 메시지

4. **새 컴포넌트**
   - `MobileNav.tsx`: 모바일 내비게이션
   - `Sheet` 컴포넌트 활용 (Shadcn UI)
